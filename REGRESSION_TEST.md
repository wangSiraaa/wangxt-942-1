# 批量价格调整回归验证测试方案

## 测试目的：验证批量价格调整后，未锁价订单价格自动重算，已付款订单价格保持锁定

---

## 测试场景 1：未锁价订单价格自动重算

### 前置条件
- 系统已启动，存在至少1个房间有基础价格（如 ¥300/晚

### 测试步骤

1. **切换到住客角色
2. **创建未锁价订单（pending状态）
   - 选择房间：2025-05-10 ~ 2025-05-12（2晚连住
   - 记录下单时价格：基础价 ¥300 × 2晚 = ¥600
   - 订单状态：pending（未付款，未锁价）
3. **记录订单号：ORDER-XXXX（pending

4. **创建已付款订单（paid状态）
   - 选择房间：2025-05-15 ~ 2025-05-17（2晚连住）
   - 下单时价格：基础价 ¥300 × 2晚 = ¥600
   - 点击"确认付款"，状态变为 paid（已付款，已锁价）
   - 记录订单号：ORDER-YYYY（paid）

5. **切换到房东角色

6. **进入"价格详情"标签页

7. **进行批量价格调整
   - 选择房间：选择上述订单所在房间
   - 日期范围：2025-05-01 ~ 2025-05-31
   - 调整方式：固定金额 +¥100
   - 点击"批量调整"

8. **预期结果
   - 弹窗提示："已自动更新 X 个未锁价订单的价格"
   - 显示具体变更列表中包含 ORDER-XXXX
   - 变更：¥600 → ¥800（(300+100）× 2晚）

9. **切换到"订单管理"标签页

10. **验证未锁价订单 ORDER-XXXX
    - 价格显示：¥800（蓝色，带 🔓 图标
    - 显示"较原价涨 ¥200"红色箭头
    - 点击查看详情：
      - 价格明细显示"（当前计算）"
      - 显示"价格已上涨"卡片
      - 原价：¥600.00
      - 当前价：¥800.00
      - 差额：+¥200.00

11. **验证已付款订单 ORDER-YYYY
    - 价格显示：¥600（灰色，带 🔒 图标
    - 无涨跌提示
    - 点击查看详情：
      - 价格明细显示"（已锁定）"
      - 无价格变动提示
      - 总价仍为 ¥600.00

## 锁价状态判定规则验证

| 订单状态 | lockedPrice | 是否锁价 | 价格变更影响 |
|---------|-------------|----------|-------------|
| pending | false | ❌ 未锁价 | ✅ 自动重算 |
| confirmed | false | ❌ 未锁价 | ✅ 自动重算 |
| paid | false | ✅ 已锁价 | ❌ 保持快照 |
| paid | true | ✅ 已锁价 | ❌ 保持快照 |
| locked | true | ✅ 已锁价 | ❌ 保持快照 |
| checkin | - | ✅ 已锁价 | ❌ 保持快照 |
| checkout | - | ✅ 已锁价 | ❌ 保持快照 |
| completed | - | ✅ 已锁价 | ❌ 保持快照 |
| cancelled | - | ✅ 已锁价 | ❌ 保持快照 |

## 核心代码验证点

### 1. 锁价判断逻辑
```typescript
// bookingStore.ts
const isLocked = order.lockedPrice || ['paid', 'locked', 'checkin', 'checkout', 'completed'].includes(order.status);
```

### 2. 价格重算触发点
- `batchUpdatePrices()` - 批量调价后
- `activatePriceVersion()` - 价格版本激活后
- `setHolidayPrice()` - 节假日价格设置后
- `setLongStayDiscount()` - 连住折扣设置后

### 3. 动态价格展示
- 未锁价订单：`getDynamicOrderPrice()` 返回当前计算价格
- 已锁价订单：`getDynamicOrderPrice()` 返回创建时快照价格
- 价格变动：`priceChanged` 标记，显示涨跌箭头和差额

---

## 事件溯源验证

### ORDER_PRICE_RECALCULATED 事件
- 事件数据包含：orderId, oldPriceSnapshot, newPriceSnapshot
- replayEvents 中正确更新订单 priceSnapshot
- 操作审计日志中记录"订单价格重算"操作

---

## 边界情况测试

1. **订单包含部分日期在调价范围内，部分不在
   - 仅调价范围内的日期使用新价格
   - 调价范围外的日期使用原价格

2. **订单跨调价生效日期
   - 仅生效日期及之后的日期使用新价格

3. **批量调价后，未锁价订单付款
   - 付款后订单变为 paid 状态，价格锁定为当前价格（已重算后的价格）

4. **批量调价后，取消未锁价订单
   - 取消时按当前价格（已重算后的价格）计算退款
