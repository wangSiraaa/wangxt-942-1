# 批量价格调整回归验证测试方案

## 测试目的
验证批量价格调整的价格基准计算正确性，以及未锁价订单价格自动重算、已付款订单价格保持锁定的完整功能。

---

## 一、价格基准计算验证（核心修复）

### 验证逻辑：批量调价必须基于「房间当前房价」计算，而非从0开始

价格优先级（从高到低）：
1. 已有节假日价（HolidayPrice）
2. 激活的价格版本（PriceVersion.active）
3. 房间基础价（Room.basePrice）

---

## 测试场景 A：普通日期 + 固定金额调价（无既有节假日价）

### 前置条件
- 房间 R1：基础价 ¥300/晚，无激活价格版本，无既存节假日价
- 测试日期：2025-06-15（普通工作日，非节假日）

### 测试步骤
1. 切换到**房东**角色
2. 进入「价格详情」标签页 → 点击「批量调整」
3. 批量调价配置：
   - 选择房间：R1
   - 日期范围：2025-06-15 ~ 2025-06-20
   - 调整方式：**固定金额**，+¥50
4. 点击「批量调整」

### 预期结果 ✅
- **每日新价 = 300 + 50 = ¥350/晚**（基准价正确使用房间基础价 ¥300）
- ❌ 错误：从 ¥0 开始加价得 ¥50/晚

### 验证方式
- 查看日历：2025-06-15 ~ 2025-06-20 每日价格应为 ¥350
- 查看操作审计日志：新节假日价记录中 price=350

## 五、核心代码验证点

### 1. 批量调价价格基准
```typescript
// priceCalculator.ts - calculateBatchPriceUpdate
// ✅ 正确：基准价三级回退
const basePrice = existing?.price ?? activeVersion?.basePrice ?? room.basePrice;

// ❌ 错误：从0开始
let newPrice = existing?.price || 0;
```

### 2. 锁价判断逻辑
```typescript
// bookingStore.ts
const isLocked = order.lockedPrice || ['paid', 'locked', 'checkin', 'checkout', 'completed'].includes(order.status);
```

### 3. 未锁价订单重算触发
- `batchUpdatePrices()` - 批量调价后
- `activatePriceVersion()` - 价格版本激活后
- `setHolidayPrice()` - 节假日价格设置后
- `setLongStayDiscount()` - 连住折扣设置后

### 4. 动态价格展示
- `getDynamicOrderPrice(orderId)`：
  - 已锁价 → 返回创建时 `snapshotPrice`
  - 未锁价 → 实时 `calculatePrice()` 计算 `currentPrice`
  - `priceChanged` 标记：`currentPrice != snapshotPrice`

## 七、边界情况测试

| # | 测试场景 | 预期结果 |
|---|---------|---------|
| 1 | 调价幅度为 0 | 不创建节假日价，不触发订单重算 |
| 2 | 降价后价格 ≤ 0 | 价格取 Math.max(0, newPrice)，即 ¥0 |
| 3 | 调价范围内无可选房间 | 提示「请选择房间和日期范围」 |
| 4 | 调价范围内没有未锁价订单 | 弹窗：「没有需要更新的未锁价订单」 |
| 5 | pending 订单调价后再付款 | 付款后锁定为**调价后**的价格 |
| 6 | 批量调价后取消未锁价订单 | 退款计算使用**调价后**的价格 |
